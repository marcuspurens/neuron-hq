# Brief: E-stop — nödavbrott via STOP-fil

## Bakgrund

Det finns idag inget sätt att avbryta en pågående Neuron-körning utan att döda hela
terminalsessionen (Ctrl+C). Det är osäkert: Ctrl+C kan lämna workspace i inkonsekvent
tillstånd, committar kanske hälfärdiga, och audit.jsonl kan trunkeras.

ZeroClaw (v0.1.7) löser detta med en STOP-fil: touch `neuron-hq/STOP` → körningen
avslutas rent vid nästa iterationsgräns.

## Uppgift

Implementera ett e-stop-mekanism i RunOrchestrator som kontrollerar om filen
`STOP` finns i repots rot före varje agent-iteration, och om den finns:
1. Loggar ett STOP-event till `audit.jsonl`
2. Skriver `STOPPED BY USER` till `runs/<runid>/report.md` (om den inte redan finns)
3. Avslutar körningen rent (returnerar med exit code 1)

## Acceptanskriterier

- [ ] `src/core/orchestrator.ts`: Kontrollerar om `STOP`-filen finns före varje
  agent-delegation (inte mitt i en iteration, utan *mellan* delegationer)
- [ ] Om `STOP` finns: loggar `{ type: "ESTOP", reason: "STOP file detected" }` till
  `audit.jsonl` och returnerar med ett tydligt felmeddelande
- [ ] CLI-utgång visar: `⛔ Run stopped by user (STOP file detected). Run ID: <id>`
- [ ] `STOP`-filen raderas INTE automatiskt — användaren måste ta bort den manuellt
  (förhindrar att nästa körning startar av misstag)
- [ ] Dokumentation i `README.md` (eller `docs/runbook.md`): hur man använder e-stop
- [ ] `tests/core/orchestrator.test.ts`: Test för att e-stop triggas korrekt
- [ ] `policy/forbidden_patterns.txt`: `STOP`-filen är undantagen från
  "skriv inte utanför workspace"-regeln (orchestrator skriver INTE filen, bara läser)

## Vad som INTE ska ändras

- Befintlig Ctrl+C-hantering (process.on SIGINT) — behålls som backup
- Workspace och run-katalog — lämnas intakta vid e-stop (ingen cleanup)
- Pågående bash-kommandon avbryts inte mitt i — e-stop triggas *mellan* delegationer

## Tekniska detaljer

**Var i koden:**
- `src/core/orchestrator.ts` — lägg till `checkEstop()` som anropas i `runAgentLoop()`
  eller motsvarande delegationsloop

**Implementationsidé:**
```typescript
private async checkEstop(runId: string): Promise<void> {
  const stopPath = path.join(process.cwd(), 'STOP');
  if (fs.existsSync(stopPath)) {
    await this.audit.log(runId, { type: 'ESTOP', reason: 'STOP file detected' });
    throw new EstopError('Run stopped by user (STOP file detected)');
  }
}
```

**Användning:**
```bash
# Starta körning i en terminal
npx tsx src/cli.ts run neuron-hq --brief briefs/...

# I en annan terminal: skapa STOP-filen för att avbryta
touch /Users/mpmac/Documents/VS\ Code/neuron-hq/STOP

# Ta bort STOP-filen efter avbrott
rm /Users/mpmac/Documents/VS\ Code/neuron-hq/STOP
```
