# TD-3c: Skills — Cross-Server Workflow Orchestration

## Problem

After TD-3a (10 servers) and TD-3b (server-scoped prompts), we still lack a way to orchestrate tools ACROSS servers. For example, "research a topic" requires aurora-search + aurora-quality + aurora-library tools. MCP Prompts are scoped to a single server. Anthropic Skills (SKILL.md files) solve this — they provide minimal-footprint workflow instructions that Claude loads on demand.

## Prerequisites

- TD-3a completed (10 scoped servers)
- TD-3b completed (server-scoped prompts)

## Solution

Create 8 SKILL.md files in `.claude/skills/` that define cross-server workflows using Anthropic's 5 design patterns. Each skill references tools from multiple MCP servers and provides step-by-step instructions.

## Technical Notes

### SKILL.md Format

Follow Anthropic's official skill format:

```yaml
---
name: researcha-amne
description: Djupforska ett ämne genom att söka i kunskapsbasen,
  identifiera luckor, föreslå forskning och generera en briefing.
  Använd denna skill när användaren vill förstå ett ämne på djupet.
---

# Researcha ämne

## När ska denna skill användas?
- Användaren frågar "vad vet vi om X?"
- Användaren vill ha en djupgående analys av ett ämne
- Användaren ber om forskningsförslag

## Steg

### 1. Sök i kunskapsbasen
Använd `aurora_search` med ämnet som query. Notera antal träffar och relevans.

### 2. Identifiera luckor
Använd `aurora_gaps` för att hitta obesvarade frågor relaterade till ämnet.

### 3. Föreslå forskning
Om luckor finns, använd `aurora_suggest_research` för att generera
konkreta forskningsförslag.

### 4. Generera briefing
Använd `aurora_briefing` med ämnet för en sammanfattande rapport.

### 5. (Valfritt) Syntetisera artikel
Om det finns tillräckligt med material, fråga användaren om de vill
skapa en syntesartikel via `aurora_knowledge_library` med action `synthesize`.

## Input
- **topic** (obligatoriskt): Ämnet att researcha
- **depth**: `quick` (steg 1-2) eller `full` (steg 1-5, standard)

## Output
Presentera en strukturerad rapport med:
- Vad vi vet (med källor)
- Vad vi inte vet (luckor)
- Föreslagna nästa steg
- Confidence-nivå för våra kunskaper
```

### Design Pattern Reference

Each skill should note which Anthropic pattern it follows:
1. **Sequential Workflow** — fixed step order, data flows between steps
2. **Multi-MCP Coordination** — orchestrates across multiple servers
3. **Iterative Refinement** — validates, fixes, re-validates
4. **Context-aware Routing** — routes based on input type
5. **Domain-specific Intelligence** — domain rules before execution

### Swedish Language

All skills should be written in Swedish (matching user preference), with tool names in their original English form.
