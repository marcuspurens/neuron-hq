# Brief: C2.1 Voiceprint Confidence Loop — lärande talaridentifiering

## Bakgrund

C2 ger manuella verktyg för att döpa om och slå ihop talare. Men Aurora borde
kunna **lära sig själv** vem som pratar — och bli bättre med tiden.

Idén: Aurora föreslår vem en talare är baserat på metadata och tidigare
bekräftelser. Användaren godkänner, redigerar eller avvisar. Varje bekräftelse
ökar confidence. Efter tillräckligt många bekräftelser (t.ex. 5) taggar Aurora
automatiskt utan att fråga.

**Flöde:**

```
Ny video ingestas
  → Diarization: SPEAKER_0, SPEAKER_1
  → Aurora kollar: "SPEAKER_0 liknar Marcus (3 bekräftelser, confidence 0.82)"
  → Föreslår: "Är detta Marcus?"
  → Användare bekräftar → confidence 0.82 → 0.86
  → Efter 5 bekräftelser (confidence ≥ 0.90): auto-taggar framöver
```

## Beroenden

- **C2 måste vara implementerad först** — rename/merge/suggest behövs som grund.

## Uppgifter

### 1. Speaker identity store: `src/aurora/speaker-identity.ts`

En ny modul som hanterar kända talaridentiteter — **separata från voice_print-noder**.
Voice prints är per-video, speaker identities är globala.

```typescript
export interface SpeakerIdentity {
  /** Unikt ID, t.ex. "speaker-marcus" */
  id: string;
  /** Visningsnamn */
  name: string;
  /** Antal gånger användaren bekräftat denna identitet */
  confirmations: number;
  /** Beräknad confidence (0–1), ökar med bekräftelser */
  confidence: number;
  /** Auto-tag threshold — tagga automatiskt när confidence >= detta */
  autoTagThreshold: number;
  /** Lista av voice_print IDs som bekräftats vara denna person */
  confirmedVoicePrints: string[];
  /** Metadata — t.ex. typisk segmentlängd, frekventa videor */
  metadata: Record<string, unknown>;
  created: string;
  updated: string;
}

/**
 * Confidence beräknas:
 *   base = 0.5 (första bekräftelsen)
 *   varje ytterligare bekräftelse: +0.1, max 0.95
 *   formel: min(0.95, 0.5 + (confirmations - 1) * 0.1)
 *
 * Auto-tag aktiveras vid confidence >= 0.90 (dvs 5+ bekräftelser)
 */
```

Lagra i Aurora-grafen som noder med `type: 'speaker_identity'`.

#### Funktioner

```typescript
/** Skapa en ny speaker identity (vid första bekräftelsen) */
export async function createSpeakerIdentity(
  name: string,
  voicePrintId: string,
): Promise<SpeakerIdentity>;

/** Bekräfta att en voice_print tillhör en känd identity */
export async function confirmSpeaker(
  identityId: string,
  voicePrintId: string,
): Promise<{ identity: SpeakerIdentity; newConfidence: number }>;

/** Avvisa ett förslag — sparar negativ signal */
export async function rejectSpeakerSuggestion(
  identityId: string,
  voicePrintId: string,
): Promise<void>;

/** Hämta alla kända identiteter */
export async function listSpeakerIdentities(): Promise<SpeakerIdentity[]>;

/** Föreslå identitet för en voice_print baserat på metadata-matchning */
export async function suggestIdentity(
  voicePrintId: string,
): Promise<Array<{
  identity: SpeakerIdentity;
  confidence: number;
  reason: string;
  autoTagEligible: boolean;
}>>;

/**
 * Auto-tag: Kör efter diarization.
 * För varje ny voice_print, kolla om en identity matchar med
 * confidence >= autoTagThreshold. Om ja, tagga automatiskt.
 * Returnerar lista av auto-taggade voice prints.
 */
export async function autoTagSpeakers(
  voicePrintIds: string[],
): Promise<Array<{
  voicePrintId: string;
  identityId: string;
  identityName: string;
  confidence: number;
  action: 'auto_tagged' | 'suggestion' | 'no_match';
}>>;
```

### 2. Matchningslogik

Matcha voice_print mot speaker_identity via:

1. **Namnmatchning** — om voice_print redan döpts om (via C2 rename) och
   namnet matchar en identity → hög confidence
2. **Video-kontext** — om identity bekräftats i samma videokälla/kanal tidigare
3. **Segment-mönster** — liknande talarlängd/frekvens (heuristik)

Framtida utökning: embedding-baserad röstmatchning via pgvector.

### 3. Integration med video-ingest

I `src/aurora/video.ts`, efter att voice_print-noder skapats:

```typescript
// Efter diarization och voice_print-skapande:
const autoTagResults = await autoTagSpeakers(newVoicePrintIds);
for (const result of autoTagResults) {
  if (result.action === 'auto_tagged') {
    // Döp om voice_print automatiskt
    await renameSpeaker(result.voicePrintId, result.identityName);
    console.log(`  🏷️  Auto-tagged: ${result.identityName} (confidence: ${result.confidence})`);
  } else if (result.action === 'suggestion') {
    console.log(`  💡 Suggestion: ${result.identityName}? (confidence: ${result.confidence})`);
  }
}
```

### 4. CLI-kommandon

#### `aurora:confirm-speaker <voicePrintId> <identityName>`

Bekräfta att en voice_print är en viss person. Skapar identity om den inte finns.

```
🎤 Confirming speaker...
  ✅ Confirmed: vp-abc-SPEAKER_0 is "Marcus"
     Identity: speaker-marcus
     Confirmations: 3 → 4
     Confidence: 0.72 → 0.82
     Auto-tag: not yet (need ≥ 0.90)
```

#### `aurora:reject-speaker <voicePrintId> <identityId>`

Avvisa ett förslag.

#### `aurora:speaker-identities`

Lista alla kända talaridentiteter med confidence.

## Avgränsningar

- **Enbart heuristisk matchning** — namnbaserad + video-kontext. Embedding-baserad
  röstmatchning (jämför röstprofiler via pgvector) planeras för C2.2.
- **Auto-tag kräver 5 bekräftelser** (konfigurerbart) — konservativt för att
  undvika feltaggning.
- **Inga notifieringar** — suggestions visas vid ingest-output och via MCP/CLI,
  inte som push-notiser.
